Linux

Requirements​

You need to install rustc and cargo. Yo can use rustup or your package manager for your system.

Cargo​

Install with cargo from crates.io

cargo install rusty-belt

Build from source​

Clone repository

git clone https://github.com/fido-node/rusty-belt.git

Build binaries with release flag

cargo build --release

Move binaries from ./target/release/ to some dir in your PATH

cp ./target/release/tmux_client ~/.local/bin
cp ./target/release/rusty_belt_server ~/.local/bin

Run as a service​

You can use this file rusty-belt.service for example:

Fix 7th line acording to the way you've installed binaries.
Use /.cargo/bin/rusty_belt_server if you choose cargo as source of binaries.
Use /.local/bin/rusty_belt_server if you build it from sources.

~/.config/systemd/user/rusty-belt.service
[Unit]
Description=Rusty-belt server part
Documentation=https://github.com/fido-node/rusty-belt

[Service]
Type=simple
ExecStart=%h/.cargo/bin/rusty_belt_server
KillMode=process
Restart=on-failure
User

[Install]
WantedBy=default.target

Place it to the ~/.config/systemd/user/

Reload services, enable and start it.

systemctl daemon-reload --user
systemctl --user enable rusty-belt.service
systemctl --user start rusty-belt.service

Use sample configs​

Use sample configs or go to configuration doc

Place these files to ~/.config/rusty-belt/

Sample app config:

~/.config/rusty-belt/config.yaml
---
server:
  update_interval: 2
segments:
  - name: left
    bg_palet:
      - f5e0dc
      - f5c2e7
      - cba6f7
      - eba0ac
      - f9e2af
      - a6e3a1
      - 94e2d5
      - 89dceb
      - 89b4fa
      - b4befe
    fg_palet:
      - 11111b
      - 181825
      - 1e1e2e
    separator: 
    parts:
      - type: session_name
        # Where v is string value with session name
        template: ": {{v}}"
      - type: vpn
        # Where v is a Vec of strings
        template: "{{#if v}}󰖂: {{#each v}}{{this}}{{#unless @last}}, {{/unless}}{{/each}}{{else}}No VPNs connected.{{/if}}"
        names:
          - substring_matcher: "10.154.1."
            name: "prod"
          - substring_matcher: "10.154.154."
            name: "dev"
      - type: mem
        # Where v is a structure with humanized string fields 'used', 'available', 'total' and 'used_percents'
        template: ": {{v.total}}/{{v.available}}/{{v.used}}"
      - type: mem
        # Where v is a structure with humanized string fields 'used', 'available', 'total' and 'used_percents'
        template: "Mem: {{v.used_percents}}% {{v.used_percents_graph}}"
      - type: cpu
        # Where v is a string value without percent symbol
        template: ": {{v.consumption}}%"
      - type: load_average
        # Where v is a structure with humanized string fields 'one', 'five' and 'fifteen'
        template: "LA: {{v.one}}, {{v.five}}, {{v.fifteen}}"
      - type: swap
        # Where v is a structure with humanized string fields 'used', 'total' and 'used_percents'
        template: "Swap: {{v.total}}/{{v.used}}"
      - type: swap
        # Where v is a structure with humanized string fields 'used', 'total' and 'used_percents'
        template: "󰾴: {{v.used_percents}}%"
      - type: disk
        dev: /dev/mapper/luks-7a504a5c-d5f0-4175-95b6-3a97c15a21ce
        # Where v is a structure with
        template: " {{v.mount_point}} {{v.used_percents}}%"
      - type: shell
        cmd: free -m -h | awk '/Mem/{printf $3"/"$2}'
        template: "{{v.stdout}}"
  - name: right
    bg_palet:
      - f5e0dc
      - f5c2e7
      - cba6f7
      - eba0ac
      - f9e2af
      - a6e3a1
      - 94e2d5
      - 89dceb
      - 89b4fa
      - b4befe
    fg_palet:
      - 11111b
      - 181825
      - 1e1e2e
    separator: 
    direction: rtl
    parts:
      - type: shell
        use_pwd: true
        cmd: gitmux -cfg ~/.config/tmux/gitmux.yaml
        template: "{{v.stdout}}"
